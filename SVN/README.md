# Subversion

> This note base on TortoiseSVN application.



## Basic Operation

### 检出

> 在服务端获取项目链接, 即可拉取项目资源文件(pull)



### 提交

> SVN没有本地暂存区, 提交直接指向服务端(提交即上传同步), 已提交的可以回到提交版本
>
> 已修改的文件可以右键还原, 每一次提交都会有记录(服务端)



### 更新

> SVN更新会将服务端资源追加到本地, 通常要先更新再提交内容



### 忽略

> - 已提交(同步), 在版本管理内的文件, 
>
>     可以右键后**去除版本控制并添加到忽略列表**(此时会有属性改变, 要提交一次)
>
> - 新添加的文件, 可以右键直接添加到忽略列表
>
> 已忽略的文件可以右键**加入**回来



### 冲突解决

> 发生冲突时, 可右键选择版本解决, 或者查看差异后自行修改内容解决



### 分支

> svn主干项目资源在**trunk**目录下, 可右键分支->创建一个新的分支出来
>
> 新创建的分支会出现在**branches**目录下, 内容同步主干项目
>
> - 此时, 可以将主干项目单独拷贝出来(移出trunk目录, 放到别处), 依然可以正常使用主干项目的svn版本控制
>
> - 同理, 可以将分支项目单独拷贝出来(移出branches目录, 放到别处), 可以正常使用分支的svn版本控制, 就像一个新项目一样
> - 当分支版本修改后, 可以右键分支项目选择合并, 选择主干项目作为合并对象实现合并
> - 右键可以在版本浏览器中自由切换版本



### 暂存

> 如需将修改的文件暂存下来, 可以右键选择**shelve**暂存, 以及取出暂存内容
